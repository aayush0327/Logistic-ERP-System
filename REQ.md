FINAL MODULES (SIMPLE & ESSENTIAL)

Each module includes only the core functionality necessary for a functional Logistics ERP.

1. SUPER ADMIN MODULE

(Used by you and your client to manage multiple logistics companies)

Functionality

Create logistics company accounts

Create company admin credentials

View and manage all tenants (companies)

Enable/disable companies

View subscription/usage (simple overview)

2. COMPANY ADMIN MODULE

(Admin dashboard for each logistics company)

Functionality

Create and manage Branch Managers, Finance Managers, Logistics Managers, Drivers

Configure company-level settings (services, prices, zones, capacity rules)

View all branches and basic reports

Manage customers (add/update customer profiles)

3. BRANCH MANAGER MODULE

(Creates sales orders based on customer requests)

Functionality

Create new sales orders (pickup, delivery, weight, quantity, truck type needed)

Edit/cancel orders (until finance approval)

Assign orders to the correct branch

Attach basic documents (invoice, packing list)

Track order status (Submitted → Finance → Logistics → Driver)

4. FINANCE MANAGER MODULE

(Approves sales orders before planning)

Functionality

View all sales orders submitted by Branch Managers

Approve / Reject orders

Add basic pricing info (if required)

Add comments if order is rejected

Mark customer payment type (COD / Prepaid)

Forward approved orders to Logistics Manager

5. LOGISTICS MANAGER MODULE

(Creates trip plans and assigns trucks/drivers)

Functionality

View all approved sales orders

Create trip plans:

Select truck and driver

Add multiple orders to one trip

Check basic truck capacity (weight/volume)

If capacity exceeds:

split into second trip OR

assign another truck

Prioritize deliveries (high/medium/low)

Approve or Reject orders for logistics reasons (e.g., no truck available)

Publish final trip plan to the Driver

6. DRIVER MODULE

(Driver mobile or web module)

Functionality

View assigned trips and delivery stops

Start trip

Update delivery status:

On the way

Arrived

Delivered

Failed delivery (with reason)

Upload Proof of Delivery (POD):

Photo

Signature (optional)

End trip

7. BASIC DASHBOARDS & REPORTING

(Simple but useful overview for each role)

Functionality

Number of orders (today, week, month)

Approved vs rejected orders

Trips created

Deliveries completed vs pending

Driver performance (simple count)

Truck utilization (basic view)

8. NOTIFICATION SYSTEM (Simple)
   Functionality

Order submitted → Finance Manager

Order approved → Logistics Manager

Trip published → Driver

Delivery completed → Branch Manager / Customer (optional)

OPTIONAL (Optional but simple add-ons)

You may choose to add these later without complexity:

Optional A: Customer Portal

Customers create pickup requests

Track shipments

Download POD

Optional B: Fleet Management Lite

Add/edit trucks

Track capacity & basic maintenance dates
